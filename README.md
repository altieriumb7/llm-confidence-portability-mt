# Multi-provider MT Confidence–Surface-Complexity Mismatch Study

This repository is an **artifact-only companion** to the paper submission. It keeps tabular and markdown evidence committed; binary figures are regenerated locally:
- `runs/aggregated/**` (final metrics/features tables)
- `paper/**` (paper-facing markdown notes and generated examples; the final LaTeX paper source is **not** included here)
- `figures/**` are regenerated by Step 4 and not versioned

The current committed study snapshot corresponds to the complete 500-sentence / 8-model run aligned with raw snapshot `runs/snapshots/20260228_000439/`. This includes GPT-5.2 / mini / nano, Claude Opus / Sonnet / Haiku, and Gemini 2.5 Pro / Flash.

## Setup
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Optional fully locked install:
```bash
pip install -r requirements.lock
```

## Environment variables
Create `.env` (or export shell vars):
- `OPENAI_API_KEY`
- `ANTHROPIC_API_KEY`
- `GEMINI_API_KEY` (fallback: `GOOGLE_API_KEY`)

See `.env.example`.

## Reproduce


## Pipeline steps
1. Build dataset:
```bash
python src/01_make_dataset.py --config configs/models.yaml
```
2. Translate + confidence:
```bash
python src/02_translate_and_confidence.py --config configs/models.yaml --input data/wmt_sample.jsonl --outdir runs/raw
```
3. Features + metrics:
```bash
python src/03_features_and_metrics.py --config configs/models.yaml --input_dir runs/raw --output runs/aggregated/dataframe.csv
```
4. Analysis + plots:
```bash
python src/04_analysis_and_plots.py --config configs/models.yaml --input runs/aggregated/dataframe.csv --outdir figures --results runs/aggregated/results_by_model.json --summary runs/aggregated/summary_table.csv --examples paper/top_mismatch_examples.md
```

## Output locations and tracking policy
- `runs/raw/` → local API cache only (**not committed**)
- `runs/aggregated/` → final aggregated artifacts (**committed**)
- `figures/` → final figures (**generated locally, not committed**)
- `paper/*` outputs → paper-facing outputs (**committed**)
- `data/wmt_sample.jsonl` is regenerated in Step 1 and is **not committed**

## API cost / rate limit note
- Step 2 makes external API calls and may incur cost.
- Providers can rate-limit requests; for reviewer runs, prefer `concurrency_per_provider: 1` in `configs/models.yaml`.

## Artifact policy and licensing note
- Committed evidence is documented in `artifacts/README.md`.
- Full `runs/raw/` caches are excluded by default for size/licensing safety and WMT text redistribution concerns.
- Tiny debugging samples are kept in `runs/raw_sample/`.
- Release-facing review should focus on `README.md`, `artifacts/README.md`, `runs/aggregated/`, `paper/`, and the source scripts under `src/`.
- Historical backup files and one-off patch files have been moved to `archive/dev_old/` so they do not appear alongside release artifacts.
